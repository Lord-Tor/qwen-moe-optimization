# 🚀 Оптимизация маршрутизации Qwen1.5-MoE [WIP]

Проект по оптимизации механизмов маршрутизации (routing) в Mixture-of-Experts (MoE) моделях для улучшения работы в конкретных доменах знаний без полного дообучения.

## 📖 Обзор проекта
Данное исследование изучает, как модификация логитов роутера в модели `Qwen1.5-MoE-A2.7B` влияет на её производительность в специфических областях (например, высшая математика MMLU). 

**Суть гипотезы:** Анализируя естественное распределение маршрутизации через `backward pass` (сбор градиентов по целевому домену), мы вычисляем наиболее релевантных экспертов. Затем, применяя частотное смещение (bias) во время инференса, мы стремимся принудительно активировать эти "доменные" нейроны, чтобы точечно улучшить точность модели.

## ⚙️ Структура репозитория
- `configs/` — JSON-файлы с векторами смещений, сгенерированные через анализ градиентов.
- `scripts/` — исполняемые Python и Slurm/Bash скрипты для инференса, оценки и анализа Jaccard Similarity.
- `src/` — основной переиспользуемый код (утилиты промптинга, BPE-токенизации, хуки роутера).
- `results/` — логи оценки и статистика маршрутизации (игнорируются в git, кроме сводных `summary`).

## 🛠 Установка
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Использование (Кластер / Локально)

**1. Генерация градиентного смещения (Domain Calibration):**
```bash
python scripts/build_gradient_bias.py --subject high_school_mathematics --output configs/math_bias.json
```
**1. Запуск инференса со смещением:**
```bash
python scripts/qwen_mmlu_biased.py --subject high_school_mathematics --bias_file configs/math_bias.json --bias_multiplier 10.0
```
**3. Массовый прогон и глубокая аналитика:**
```bash
sbatch scripts/run_night.sh
python scripts/analyze_all.py
```

## 📊 Текущие результаты и инсайты
*Ловушка токенизации и устойчивость роутера:
В ходе экспериментов на кластере с dual TITAN RTX была выявлена и решена проблема несовместимости префиксов BPE-токенизатора tiktoken.
Первые стресс-тесты с агрессивным множителем логитов (x100.0) показали перекрытие маршрутов (Jaccard Overlap) на уровне ~32%. Это математически подтверждает работоспособность наших Forward-хуков — роутер действительно меняет выбор экспертов. Текущая задача — собрать чистые метрики Delta Accuracy на исправленных BPE-токенах.
**Базовая оценка (MMLU Baseline):**
* **Домен:** `high_school_mathematics`
* **Выборка:** Полный тестовый сплит (270 примеров)
* **Метод:** One-pass next-token logprob scoring
* **Точность:** 29.26% (79/270)
